GAME·TAP BIBLE ANIMATION FRAMEWORK v1

PURPOSE:
Create simple, clean, reusable SVG animations for literacy/story lessons.
Animations must be lightweight, readable, child-friendly, and modular.

PRIMARY USE CASE:
David throws a stone at Goliath.

==================================================
1. GLOBAL STYLE RULES
==================================================

Visual Style:
- Flat 2D SVG
- Clean shapes
- Rounded corners where possible
- Bold readable silhouettes
- Minimal detail
- Child-friendly proportions
- No realism
- No scary faces
- Consistent line thickness

Colour Palette:
- Earth tones
- Warm sand / brown / green / blue
- Accent gold for heroes
- Accent red for danger
- Max 6 colours per scene

Scale Rules:
- David = base scale 1.0
- Soldier = 1.2
- Goliath = 2.2 to 2.8
- Props proportional to character

Animation Rules:
- Smooth easing
- 1 to 3 seconds typical actions
- Idle motion subtle only
- No chaotic motion
- Max 2 simultaneous animated elements

Performance Rules:
- Pure SVG + CSS/JS
- No libraries required
- Mobile friendly
- GitHub Pages compatible

==================================================
2. CORE ASSET PACK
==================================================

CHARACTERS:

DAVID:
- shepherd boy
- tunic
- sling
- simple hair
- confident stance

GOLIATH:
- giant warrior
- helmet
- shield optional
- broad stance
- intimidating silhouette

CROWD:
- simple background silhouettes

PROPS:

- stone
- sling strap
- shield
- hill
- battlefield ground
- sky backdrop
- dust puff

==================================================
3. RIGGING RULES (IMPORTANT)
==================================================

Each character built from separate SVG groups:

David:
- head
- torso
- left arm
- right arm
- left leg
- right leg
- sling hand
- back arm

Goliath:
- head
- torso
- arms
- legs

This allows transforms / animation by part.

==================================================
4. ANIMATION GRAMMAR
==================================================

Allowed reusable actions:

IDLE_BREATHE:
small scale pulse

STEP_FORWARD:
translate x

ARM_RAISE:
rotate arm upward

SLING_SPIN:
rotate sling hand circularly

RELEASE:
stone detaches from sling

PROJECTILE_ARC:
stone moves in curve

IMPACT:
target head snaps back + shake

FALL:
rotate + drop

CHEER:
crowd bounce

==================================================
5. CAMERA / LAYOUT RULES
==================================================

Scene size:
16:9 responsive

Foreground:
David left side

Midground:
stone path

Right side:
Goliath

Background:
hills / sky

==================================================
6. DAVID VS GOLIATH SCENE BUILD
==================================================

TIMELINE:

0.0 - 1.0
Idle scene
David left, Goliath right

1.0 - 2.2
David raises sling arm

2.2 - 3.2
Sling spins

3.2
Release stone

3.2 - 4.0
Stone arcs toward Goliath

4.0
Impact on forehead

4.0 - 5.5
Goliath shakes then falls

5.5 - 7.0
Victory pause

==================================================
7. CLAUDE BUILD RULES
==================================================

When generating scenes:

- Reuse existing assets first
- Do not redraw characters each prompt
- Build modular SVG groups
- Use transforms, not frame-by-frame paths
- Keep code readable
- Separate CSS animation logic
- Label IDs clearly

==================================================
8. OUTPUT FORMAT
==================================================

Claude should output:

- single HTML file OR SVG + CSS + JS
- self-contained
- responsive
- editable
- comments for asset sections
